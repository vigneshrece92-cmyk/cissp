What is CISSP?

The Certified Information Systems Security Professional (CISSP) is the gold standard of information security certifications globally. Directed by senior leadership and executives, the CISSP is not a technical configuration exam; it is a risk management, governance, and business-focused assessment designed to validate your capability to architect, design, and manage an organization's security posture.

Holding the credential is a gateway to leadership roles like Chief Information Security Officer (CISO), Security Director, Security Architect, and Lead Consultant.

Certification Requirements

  • Professional Experience: 5 years of cumulative paid work experience in 2 or more of the 8 domains.
  • Degree Waiver: 1-year waiver available for holding a 4-year college degree or a recognized credential (e.g. Security+, CISA, CEH).
  • Endorsement: Pass the exam and receive endorsement from an active (ISC)² member in good standing within 9 months.

Exam Structure & CAT Mechanics (2026 Update)

The English CISSP exam uses Computerized Adaptive Testing (CAT). The testing engine adjusts dynamically based on your performance, ranging from 100 to 150 questions over a maximum of 3 hours. Crucially, backtracking is disabled: you must submit each answer to proceed, and cannot review or change previous answers. A passing standard of 700/1000 is required.

Elite Manager-Level Exam Tips

The CISSP is a management exam. Keep these strategic rules in mind when answering questions:

Think Like a Manager

You are an advisor/risk consultant, not a technician. Do not fix problems yourself. Look for policy, business continuity, or process solutions.

Safety First

Human safety is always the absolute priority. If any question involves life safety, select the option that protects human life first.

Identify Qualifiers

Search for keywords like MOST, LEAST, BEST, FIRST, or NOT. These words completely change what the question is asking.

Security is an Enabler

Security is not there to restrict the business; it is there to support and enable business objectives. Align security goals with leadership.

Core Security Access Models

Security Model Primary Focus Core Principles Key Axiom / Properties
Bell-LaPadula Confidentiality Prevents unauthorized disclosure of classified details. Lattice-based MAC. No Read Up (Simple Security), No Write Down (*-Property).
Biba Integrity Prevents unauthorized modification of files. Lattice-based MAC. No Read Down (Simple Integrity), No Write Up (*-Integrity).
Clark-Wilson Integrity Transactions are validated via constrained processes and separation of duties. Access Triple (Subject, Program, Object). Transformation Procedures (TPs).
Brewer-Nash Confidentiality Prevents conflict of interest in consultancy environments dynamically. Chinese Wall (Restricts access based on history).

CISSP Ultimate Cheat Sheet (Quick Reference)

A distilled summary of essential exam concepts, formulas, ciphers, and regulatory standards.

Category / Concept Core Formula / Principle Exam Context & Key Details
Risk Calculations SLE = AV × EF
ALE = SLE × ARO		 Asset Value (AV) is cost, Exposure Factor (EF) is % loss. ALE represents the annual loss expectation. Helps determine if control cost is justified.
Symmetric Keys Keys = n × (n - 1) / 2 Calculates the number of symmetric keys required for n users to communicate securely. Symmetric is fast, but key distribution does not scale.
Digital Evidence Order of Volatility Collect most volatile data first: 1. CPU Registers/Cache → 2. RAM → 3. Network States/Routing Tables → 4. Disk → 5. Backups.
Cryptosystems Kerckhoffs' Principle The security of a cryptosystem must rest entirely on the secrecy of the key, not on the secrecy of the algorithm (no security by obscurity).
Security Models Confidentiality vs. Integrity Bell-LaPadula: No Read Up, No Write Down (confidentiality).
Biba: No Read Down, No Write Up (integrity).
IPsec Encapsulation ESP vs. AH ESP: Encrypts payload (Confidentiality + Integrity).
AH: Signatures only (Integrity, no confidentiality; does not encrypt).

27002: Specific controls.
27005: Risk management.
27701: Privacy extension.
Privacy Mandates GDPR / HIPAA / GLBA GDPR: EU rights, 72-hour breach notify.
